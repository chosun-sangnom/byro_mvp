## 2. 입력 방식

- 모든 항목의 기본 입력은 **수동**이다.
- 경력·학력은 수동 입력 후 별도 인증 버튼으로 인증 배지를 획득할 수 있다.
- URL 입력: `article-interview`(기사/인터뷰)만 지원. 입력된 URL로부터 썸네일 자동 스크래핑.

## 4. 인증 정책

경력(`career-role`)과 학력(`education-history`)만 인증 지원. 인증되지 않은 항목은 배지 없이 그대로 노출.

### 4-1. 경력 인증

**흐름**
1. 경력 인증 버튼 탭
2. 카카오 본인인증 (실명 + 생년월일 확인)
3. 건강보험공단 API 호출 → 직장가입자 이력 자동 조회
4. 조회된 직장 목록 미리보기 → 항목별 포함/제외 선택
5. 확정 후 경력 항목으로 자동 import, 인증 배지 부여

**예외 처리**

| 상황 | 처리 |
|------|------|
| API 조회 실패 | 실패 안내 후 수동 추가 (인증 배지 없음) |
| 건강보험공단 미가입 (프리랜서 등) | 수동 추가 (인증 배지 없음) |
| 이직 후 재인증 | 수동 재인증 (자동 갱신 없음) |

### 4-2. 학력 인증

두 가지 방법 중 선택.

**방법 A — 졸업증명서 업로드 (OCR)**
1. 졸업증명서 PDF 또는 이미지 업로드
2. OCR로 문서 파싱 → 실명 + 생년월일 추출
3. 가입 시 등록된 실명·생년월일과 대조
4. 일치 시 인증 완료, 인증 배지 부여

**방법 B — 학교 이메일 인증**
1. 학교 발급 이메일 주소 입력 (예: `hong@korea.ac.kr`)
2. 해당 이메일로 6자리 인증코드 발송 (유효 시간 10분)
3. 코드 입력 → 일치 시 인증 완료

**세부 정책**

| 항목 | 정책 |
|------|------|
| 인증 범위 | 대학 ~ 대학원 (복수 인증 가능) |
| 이메일 인증 지원 대상 | 대학교·대학원만 (고등학교는 OCR만) |
| OCR 파싱 오류 | 운영자 수동 검토 (추후 백오피스) |
| 업로드 서류 보존 | 인증 완료 즉시 파기 |
| 인증 유효 기간 | 무기한 유지 |
| 수동 입력 후 OCR 인증 시 | 병합 처리 (블록 2개 생성 안 함) |

## 5. 공개 프로필 표시 정책

하이라이트는 공개 프로필 **나** 탭의 하이라이트 섹션에 표시.

### 5-1. 그룹 표시 순서

커리어 → 업적 → 라이프스타일 순으로 렌더링. 항목이 하나도 없는 그룹은 섹션 자체 숨김.

### 5-2. 수동 항목 그룹 미리보기

- 카테고리 내 항목 1개: 해당 항목 제목을 그대로 미리보기로 표시
- 카테고리 내 항목 2개 이상: **대표 항목** 제목 + `외 N개` 형식
- 탭 클릭 시 카테고리 내 전체 항목 확장 (Accordion)

### 5-3. 대표 하이라이트

| 항목 | 정책 |
|------|------|
| 기본값 | `metadata.isPrimary === true`로 표시된 항목 |
| 오너 재지정 | 내 Byro → 하이라이트 관리 화면에서 변경 가능 |
| 미지정 시 | 목록 첫 번째 항목을 자동 대표로 사용 |
